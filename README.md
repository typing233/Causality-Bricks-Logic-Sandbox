# 🧱 Causality Bricks - 3D 积木因果逻辑平台

一个将 3D 积木搭建与可视化因果逻辑相结合的互动解谜平台。玩家可以通过为积木编写物理规则来创造动态运转的精妙机关。

## ✨ 核心特性

### 🎮 3D 物理沙盒
- **多种几何体**：支持立方体、球体、圆柱体、平面等基础几何体
- **真实物理引擎**：基于 Cannon.js 的真实物理模拟
- **拖拽选择**：点击选择物体，实时查看和编辑属性

### ⚖️ 物理属性编辑
- **质量调节**：0.1kg - 50kg 可调节
- **弹力控制**：0% - 100% 弹力系数
- **摩擦系数**：精细调节表面摩擦力
- **颜色自定义**：完整的颜色选择器和预设调色板
- **固定/动态**：切换物体是否受物理影响
- **重力开关**：单独控制每个物体的重力

### 📜 因果规则系统
通过下拉菜单设定基础的因果事件，例如：
- **触发条件**：
  - 碰撞事件：当物体 A 碰撞物体 B 时
  - 接近事件：当物体 A 接近物体 B 时
  - 速度阈值：当物体速度超过指定值时
  
- **执行动作**：
  - 关闭重力
  - 开启重力
  - 设为固定物体
  - 设为动态物体
  - 施加向上冲量
  - 随机改变颜色

### 🏆 内置物理挑战

1. **悬浮小球**（简单）
   - 任务：让小球在不接触地面的情况下悬停 5 秒
   - 提示：创建一个规则，当球体碰撞触发器时，关闭球体的重力

2. **连锁反应**（中等）
   - 任务：创建一个多米诺骨牌式的连锁反应，触发至少 5 个物体
   - 提示：利用碰撞规则和重力控制实现复杂的连锁效果

3. **完美着陆**（中等）
   - 任务：让球体从高处落下，精确落在指定平台上
   - 提示：利用规则在球体接近平台时关闭重力

## 🚀 快速开始

### 环境要求
- Python 3.8+
- pip 包管理器

### 安装步骤

1. **克隆项目**（如果需要）
```bash
cd Causality-Bricks-Logic-Sandbox
```

2. **创建虚拟环境**（推荐）
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **启动服务**
```bash
python main.py
```

5. **访问平台**
打开浏览器访问：`http://localhost:8234`

## 🎯 使用指南

### 基本操作

1. **添加物体**
   - 点击左侧侧边栏的「物体」标签
   - 选择想要添加的几何体（立方体、球体等）
   - 调整生成设置（质量、弹力、颜色等）
   - 点击形状按钮即可在场景中生成物体

2. **选择物体**
   - 在 3D 场景中点击物体即可选中
   - 选中后，点击「属性」标签查看详细属性
   - 可以实时调整位置、缩放、物理属性等

3. **播放/暂停**
   - 点击工具栏的「播放/暂停」按钮
   - 或按 `空格键` 快速切换
   - 播放时物体会遵循物理规则运动

4. **重置场景**
   - 点击「重置」按钮或按 `R` 键
   - 所有物体将恢复到初始位置和状态

### 创建因果规则

1. **切换到规则面板**
   - 点击侧边栏的「规则」标签

2. **添加新规则**
   - 点击「+ 添加新规则」按钮

3. **配置规则**
   - **源物体**：选择触发规则的物体 A
   - **触发类型**：选择触发条件（碰撞、接近、速度）
   - **目标物体**：选择被影响的物体 B
   - **动作类型**：选择要执行的动作

4. **启用/禁用规则**
   - 点击规则卡片上的「启用/禁用」按钮

5. **删除规则**
   - 点击规则卡片上的「删除」按钮

### 场景保存与加载

1. **保存场景**
   - 点击「物体」标签下的「保存」按钮
   - 输入场景名称，点击确认
   - 场景将保存到服务器的 `saves/` 目录

2. **加载场景**
   - 点击「加载」按钮
   - 选择要加载的场景
   - 场景将自动恢复到保存时的状态

3. **清空场景**
   - 点击「清空」按钮
   - 确认后将删除当前场景中的所有物体和规则

### 挑战模式

1. **选择挑战**
   - 点击「挑战」标签
   - 从列表中选择一个挑战

2. **完成挑战**
   - 系统会自动加载预设的场景
   - 利用规则系统完成挑战目标
   - 悬浮挑战会显示实时计时器

3. **成功反馈**
   - 完成挑战后会显示成功弹窗
   - 物理模拟会自动暂停

## 📡 API 接口

后端使用 FastAPI 提供以下 RESTful API：

### 场景管理

#### 获取场景列表
```http
GET /api/scenes
```
返回所有已保存的场景信息。

#### 获取单个场景
```http
GET /api/scenes/{name}
```
返回指定场景的完整数据。

#### 保存场景
```http
POST /api/scenes/{name}
Content-Type: application/json

{
    "name": "场景名称",
    "objects": [...],
    "rules": [...]
}
```
保存场景数据。

#### 删除场景
```http
DELETE /api/scenes/{name}
```
删除指定场景。

### 挑战系统

#### 获取挑战列表
```http
GET /api/challenges
```
返回所有可用的挑战信息。

### 健康检查

```http
GET /health
```
检查服务是否正常运行。

### API 文档

启动服务后，可以访问：
- Swagger UI: `http://localhost:8234/docs`
- ReDoc: `http://localhost:8234/redoc`

## 🐛 故障排除

### 常见问题

1. **服务无法启动**
   - 检查 Python 版本是否 >= 3.8
   - 确认依赖已正确安装
   - 检查 8234 端口是否被占用

2. **3D 场景不显示**
   - 检查浏览器控制台是否有错误
   - 确认 Three.js 脚本正确加载
   - 尝试刷新页面

3. **物理模拟异常**
   - 检查物体质量设置
   - 确认碰撞体大小与模型匹配
   - 调整时间步长参数

4. **场景保存失败**
   - 检查 `saves/` 目录权限
   - 确认磁盘空间充足
   - 查看后端控制台错误信息
